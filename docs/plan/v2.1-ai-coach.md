# v2.1 — AI Coach Mode

## Goal

AI doesn't play — it watches and advises.
Shows suggestions, explains reasoning, reviews replays after loss.

---

## Modes

| Mode | Description |
|---|---|
| **Autoplay** | AI controls everything (v2.0) |
| **Coach** | AI suggests next move, user clicks to confirm |
| **Explain** | AI explains why current board state is good/bad |
| **Replay Review** | After game over, AI analyzes what went wrong |

---

## Coach Mode

### How It Works

1. AI watches game state every tick
2. Shows suggestion overlay on board
3. User can accept (click) or ignore
4. AI adapts suggestion based on user's choices

### Suggestion Display

```
┌─────────────────────────────────────┐
│  💡 AI Coach suggests:              │
│  Place FIREWALL at row 2, col 3    │
│  Reason: Keylogger approaching     │
│          with no defense in lane   │
│                                     │
│  [Accept] [Ignore] [Don't ask 10s] │
└─────────────────────────────────────┘
```

### Visual Highlight

- Suggested cell: glowing cyan border
- Arrow from suggestion to threat
- Pulsing animation until accepted/ignored

### Implementation

```js
const COACH_CONFIG = {
  enabled: false,
  suggestInterval: 3000,
  highlightDuration: 5000,
  autoSuggest: true,
  showReasoning: true
};

function coachTick() {
  if (!COACH_CONFIG.enabled) return;

  const state = getState();
  const legalActions = getLegalActions();
  const suggestion = getAISuggestion(state, legalActions);

  if (suggestion) {
    showCoachOverlay(suggestion);
  }
}

function getAISuggestion(state, legalActions) {
  // Use rule bot logic but return suggestion instead of action
  const actions = ruleBotDecide(state, legalActions);
  if (actions.length > 0) {
    return {
      action: actions[0],
      reason: generateReason(state, actions[0]),
      highlight: getCellCoords(actions[0])
    };
  }
  return null;
}

function generateReason(state, action) {
  switch (action.type) {
    case 'place_tower':
      const threats = state.lanes[action.row]?.threats || [];
      if (threats.length > 0) {
        return `${threats[0].type} approaching with no defense in lane`;
      }
      return `Build economy for later waves`;
    case 'upgrade_tower':
      return `Upgrade increases DPS for tougher threats ahead`;
    case 'sell_tower':
      return `This tower is not contributing, free up credits`;
    case 'send_wave':
      return `Board is stable, safe to start next wave`;
    default:
      return `Waiting for better opportunity`;
  }
}
```

## Explain Mode

### How It Works

User clicks "Explain" button, then clicks any cell/tower/threat.
AI explains the current state of that element.

### UI Flow

```
1. User clicks [Explain] button
2. Cursor changes to question mark
3. User clicks on:
   - Empty cell → "Good spot for a tower because..."
   - Tower → "This tower is effective because..."
   - Threat → "This threat is dangerous because..."
   - Fog area → "This area is foggy, place Scanner to reveal"
4. AI response shown in panel
```

### Implementation

```js
function explainCell(row, col) {
  const state = getState();
  const cell = state.lanes[row]?.cells[col];

  if (!cell) return 'Invalid cell';

  if (cell.hasTower) {
    return explainTower(cell);
  } else if (cell.type === 'water') {
    return 'Water cell — place PROXY_NODE first, then build on it';
  } else if (cell.type === 'grave') {
    return 'Grave cell — threat spawn point, cannot build here';
  } else {
    return explainEmptyCell(row, col, state);
  }
}

function explainTower(cell) {
  const tower = TOWER_TYPES[cell.towerKey];
  const threats = getCurrentThreats();

  const effective = threats.filter(t =>
    tower.damage >= t.hp * 0.3
  );
  const weak = threats.filter(t =>
    t.speed > 0.3 || t.special === 'cloaked'
  );

  return {
    name: tower.name,
    stats: `DMG: ${tower.damage} | Rate: ${tower.fireRate}s | Range: ${tower.range}`,
    effective: `Effective against: ${effective.map(t => t.name).join(', ') || 'none'}`,
    weak: `Weak against: ${weak.map(t => t.name).join(', ') || 'none'}`,
    suggestion: generateUpgradeSuggestion(cell)
  };
}

function explainEmptyCell(row, col, state) {
  const lane = state.lanes[row];
  const threats = lane.threats;
  const towers = lane.towers;

  if (threats.length > 0 && towers.length === 0) {
    return `URGENT: Place a shooter here! ${threats[0].type} is approaching with no defense.`;
  }

  if (towers.length >= 3) {
    return `Lane is well-defended. Consider upgrading existing towers instead.`;
  }

  if (state.credits < 100) {
    return `Not enough credits for most towers. Collect tokens or wait.`;
  }

  return `Good spot for a tower. Consider: ${recommendTower(row, state)}`;
}
```

## Replay Review Mode

### How It Works

After game over (loss), show "AI Review" button.
AI analyzes the full game state at time of loss.
Provides structured feedback.

### Review Output

### Implementation

```js
function generateReplayReview(gameState) {
  const analysis = {
    waveReached: gameState.wave,
    creditsEarned: gameState.totalCreditsEarned,
    towersPlaced: gameState.towersPlaced,
    threatsKilled: gameState.threatsKilled,
    problems: [],
    keyMoments: [],
    suggestions: []
  };

  // Analyze lane coverage
  for (let row = 0; row < GRID_ROWS; row++) {
    const laneThreats = gameState.threatHistory.filter(t => t.row === row);
    const laneTowers = gameState.towers.filter(t => t.row === row);

    if (laneThreats.length > 0 && laneTowers.length === 0) {
      analysis.problems.push(`Lane ${row} had no defense`);
    }
  }

  // Analyze tower types
  const producers = gameState.towers.filter(t => TOWER_TYPES[t.key].type === 'producer');
  const shooters = gameState.towers.filter(t => TOWER_TYPES[t.key].type === 'shooter');

  if (producers.length > shooters.length) {
    analysis.problems.push('Too many producers, not enough DPS');
  }

  // Generate suggestions
  if (analysis.problems.length > 0) {
    analysis.suggestions.push('Focus on lane coverage before economy');
  }

  return analysis;
}
```

## Implementation Notes

- Coach mode: overlay div positioned over grid cells
- Explain mode: click handler on grid, show response in side panel
- Replay review: store game history events in array, analyze on game over
- LLM explanations: use same provider as v2.0 with coach-specific prompt
- Rule-based explanations: no API needed, use template strings
- All modes can work without LLM (rule-based fallback)
